# h4

## Tiivistelmä

4.12.1 Size and Complexity of Some DSLs

  - Järjestelmät luovat omia epästandardeja konseptejaan, mikä lisää käyttäjän kognitiivista kuormaa.
  - DSL:t voivat olla hyvin erilaisia ja monimutkaisia, vaikka niiden tarkoitus on rajattu tiettyyn käyttöön

4.12.2 Use of DSL Functions in Case Configuration

  - Vaikka Puppetissa ja muissa CM työkaluissa on paljon toimintoja, käytännössä vain pieni osa niistä kattaa suurimman osan käytöstä
  - Suurin osa kaikesta konfiguraatiokoodista rakentuu neljän perusasian eli tiedostojen, pakettien, palveluiden ja komentojen suorittamisen ympärille.
  - Analyysi osoittaa, että vaikka DSL on laaja, todellisessa käytössä toiminnot keskittyvät harvoihin peruskomentoihin

4.12.3.1 Dependencies Between Main Functions

  - Kaikki konfiguraationhallinnan monimutkaiset operaatiot voidaan pohjimmiltaan pelkistää joko tiedostojen muokkaamiseen tai ulkoisten komentojen ajamiseen.
  - Järjestelmän toiminta perustuu siihen, että se tarkistaa resurssin nykytilan ja suorittaa toimenpiteitä ainoastaan silloin, kun havaittu tila poikkeaa määritellystä tavoitteesta.
  - Lähes kaikki konfiguraatiotarpeet voidaan rakentaa muutaman perustoiminnon varaan, mikä pitää järjestelmän ytimen yksinkertaisena.
