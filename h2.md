# H2

## Karvinen 2026: Sudo without password

- Ennen muokkauksia on avattava toinen ikkuna, jossa on valmiina pääkäyttäjän istunto.
- Kannattaa luoda salasanaton sudoless ryhmä, johon voi märittää sudovapaan toiminnon.
- %sudoless ALL = (ALL) NOPASSWD: ALL antaa ryhmälle täydet oikeudet ilman salasanaa.
- Komenolla sudo visudo /etc/sudoers.d/sudoless komentoa uuden säännön kirjoittamiseen.
- toimivuutta testattiin komennolla sudo -k jälkeen, mikä vahvistaa pääsyn ilman salasanakyselyä.
  
## Karvinen 2026: Passwordless Sudo with Ansible

- Luodaan järjestelmään uusi käyttäjä ja oma ryhmä käyttäen vahvaa salasanaa. 
- Ennen asetusten muuttamista avataan valmiiksi erillinen pääkäyttäjän istunto.
- /etc/sudoers.d/-hakemistoon uusi sääntötiedosto käyttämällä visudo työkalua. Tiedostoon kirjoitetaan %sudoless ALL = (ALL) NOPASSWD: ALL
- Kirjaudutaan sisään uudella käyttäjällä ja nollataan sudolupa

## Tehtävä C

<img width="955" height="683" alt="image" src="https://github.com/user-attachments/assets/4bda5be0-987e-4fe6-ba08-2d43d6334660" />

Loin uuden kimians-hakemiston. 
Sen sisällä määrittelin roolin, joka luo sudoless ryhmän, lisää kimians käyttäjän tarvittaviin ryhmiin, asettaa SSH avaimen ja antaa ryhmälle oikeuden suorittaa komentoja ilman sudo salasanaa. Lopuksi tarkistin hakemistorakenteen komennolla tree -F.

<img width="1113" height="698" alt="image" src="https://github.com/user-attachments/assets/329e8b27-1754-41db-971b-227f4acbff8b" />

Kohdassa - name: pitäisi alkaa taski, mutta jätin tietämättä ihan tyhjäksi. Sen alla on apt moduuli, jolla on tarkoitus asentaa kaksi pakettia: htop ja curl.
state: present tarkoittaa sitä, että Ansible varmistaa niiden löytyvän koneelta, ja update_cache: yes ajaa sen perus pakettilistojen päivityksen.

Kun koko settiä ajaa ansible-playbook site.yml komennolla, niin se pamahtaa heti virheeseen. Virhe johtui sisennyksestä: - name oli sijoitettu edellisen tehtävän sisään, jolloin Ansible ei löytänyt sille moduulia.

## TEHTÄVÄ D


<img width="960" height="685" alt="image" src="https://github.com/user-attachments/assets/aa59222c-6680-4d9a-a2d5-8d607208a830" />

Loin loppuun uuden tehtävän nimeltä "Tiedosto". Käytin tässä copy-moduulia, jolla pystyin kirjoittamaan tekstiä suoraan tiedostoon ilman erillisiä lähdetiedostoja.
Asetin tiedoston kohdepoluksi /home/kimians/kimiteksti.txt. Näin varmistin, että tiedosto luodaan suoraan luodun käyttäjän kotihakemistoon.
Content osiossa käytin pystyviivaa. Sen avulla pystyin kirjoittamaan tiedoston sisällön usealle eri riville. 
Asetin molempiin arvoksi kimians. Tarkoittaen sitä, että pelkästään sillä käyttäjällä pääsen tarkistamaan tiedostoa. 

0 = Ensimmäinen nolla tarkoittaa, ettei tiedostolle ole asetettu erikoisbittejä.

6 = Tämä on tärkein luku. Se on summa numeroista 4 ja 2 eli 6 = luku + kirjoitus.

0 = Nolla tarkoittaa, ettei ryhmän jäsenillä ole mitään oikeuksia tiedostoon.

0 = Myöskään kenelläkään muulla järjestelmän käyttäjällä ei ole mitään pääsyä tiedostoon.
